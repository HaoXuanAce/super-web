# 五、前端推荐结构

```ts
type FilterOption = {
  id: string
  name: string
  category: string
  scene: 'portrait' | 'landscape' | 'selfie' | 'ecommerce'
}

type FilterGroup = {
  category: string
  label: string
  options: FilterOption[]
}
```

前端只保存：

```json
{
  "id": "portrait_cinematic",
  "name": "电影感人像",
  "category": "电影人像",
  "scene": "portrait"
}
```

后端维护：

```json
{
  "id": "portrait_cinematic",
  "prompt": "将整张人像照片处理为电影感风格……",
  "negativePrompt": "禁止改变人物身份……",
  "defaultStrength": 0.35
}
```

# 六、后端拼接顺序

```text
1. 图片类型基础约束
2. 用户选择的滤镜提示词
3. 用户选择的滤镜强度
4. 图片内容保留要求
5. 对应类型的负向提示词
```

推荐最终结构：

```text
请基于用户上传的原图进行整体风格处理。

图片类型：
{portrait / landscape / selfie / ecommerce}

保留要求：
{该类型的基础保留要求}

所选滤镜：
{根据 id 获取到的 prompt}

滤镜强度：
{轻微 / 自然 / 明显 / 强烈}

负向约束：
{该类型的 negativePrompt}
```
